Test Case INIT-0010
Title Initialization with no existing SQLite database
Objective

This test case is intended to verify the following implied or specified requirements:

  • When the app is started by selecting from the Android app selection menu, the app shall do the following:
  • If the portfolio database does not exist, the app shall create the database and then display a blanked main menu.
  • rtnv shall initialize and create an empty portfolio if none exists (first time run on the device).
  • rtnv shall use SQLite to create a persistent store of the portfolio on the target device.

The objective of this test case is to verify the above requirements. This test will be executed the first time the app is loaded onto the Android device. At that point, no portfolio will exist so the image of it, the SQLite database, will be created and will contain no stock entries.
Setup
  • Android device with version 4.2 or greater is attached to the Android development system. There is no copy of rtnv installed on the device.
Actions
  1. Compile and load rtnv onto the target Android device.
  2. Verify the compile completes with no errors.
  3. Verify the installation completes with no errors.
  4. Verify that a screen is displayed that has a text field for the ticker symbol, a text field for the number of shares, and a text field for the value of each share.
  5. Verify that there are four buttons: Enter, Delete, Next, and Calculate Net Value.
  6. Verify that no runtime errors are splashed to the screen.
Expected With this being the first time the app has run on the device, the database should be created and the main menu should be void of valid stock entries.
Results PASSED
Cleanup
  • Unnecessary
Requirements RTNV-PR-INIT-010, RTNV-PR-INIT-010.001, RTNV-PROV-010, RTNV-EXT-IF-070
