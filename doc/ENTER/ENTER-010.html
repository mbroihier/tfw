Test Case ENTER-010
Title Pressing Enter when there is a Valid Ticker Symbol and Valid Number of Shares
Objective

This test case is intended to verify the following implied or specified requirements:

  • When the enter button is pressed, rtnv shall send a retrieve request to the public server if the ticker field is not empty.
  • rtnv shall store a persistent copy of the ticker, number of shares, and value of each share in the persistent database.
  • rtnv shall clear the ticker, number of shares, and value of each share fields at completion of processing.

The objective of this test case is to verify the requirements identified above which cover the normal entry of a stock into the portfolio.
Setup
Actions
  1. Delete the app from the target device.
  2. Compile and load rtnv onto the target Android device.
  3. Verify the compile completes with no errors.
  4. Verify the installation completes with no errors.
  5. Select the "Ticker Symbol" field and enter the ticker value of AAPL.
  6. Select the "Number of Shares" field and enter the value 200.
  7. Press the "Enter" button.
  8. Verify a "blank" sceen is displayed.
  9. Select the "Ticker Symbol" field and enter the ticker value of INTC.
  10. Select the "Number of Shares" field and enter the value 500.
  11. Press the "Enter" button.
  12. Verify a "blank" screen is displayed.
  13. Press the "Next" button and verify that the symbol AAPL is displayed with 200 shares and that a non zero value is displayed for the last closing price of the stock.
  14. Press the "Next" button again and verify that the symbol INTC is displayed with 500 shares and that a non zero value is displayed for the last closing price of the stock.
  15. Press the "Next" button again and verify that a "blank" screen is displayed.
Expected With normal entry of a stock into the portfolio, the stock will be entered, the public server will retrieve the last closing price, and the stock will stored into the persistent database. The pressing of "Next" will illustrate that the stock was read from the database and that the public query will have updated the closing value.
Results PASSED
Cleanup
